## 2. 詳細内容説明

### 1.1. GitHub Copilot Agent Skillsとは何か

**説明内容：**
- Agent Skillsの定義：Copilotに教え込む再利用可能な手順書
- スキルが解決する課題
- 従来手法（毎回同じプロンプトを書く）との違い
- スキルのライフサイクル

**出力形式：**
- 概念図（Mermaid, SVG）
- 図解+テキスト説明

---

### 1.2. 従来のプロンプト手法との違い

**比較項目：**
| 項目 | 従来（毎回プロンプト） | Agent Skills |
|------|----------------------|-------------|
| 再利用性 | 低 | 高 |
| 学習曲線 | 短 | 中 |
| スケーラビリティ | 低 | 高 |
| チーム共有 | 困難 | 容易 |
| カスタマイズ | 毎回 | 初回のみ |

---

### 1.3. スキルの仕組み理解

**トピック：**
- スキル定義ファイルの構造
- メタデータ（名前、説明、パラメータ）
- 実行フロー
- Copilotとの相互作用

**コード例：**
- スキル定義ファイル（JSON/YAML）のサンプル

---

### 2.1. MCP（Model Context Protocol）との違い

**比較観点：**
- **目的の違い**
  - Agent Skills：手順書の再利用
  - MCP：LLMが外部ツール・データに安全にアクセス
  
- **アーキテクチャの違い**
  - Agent Skills：Copilot内蔵メカニズム
  - MCP：汎用プロトコル（複数のLLMで使用可能）

- **使い分けシナリオ**
  - Agent Skillsの方が適しているケース
  - MCPの方が適しているケース
  - 両方を組み合わせるケース

**表現方法：**
- Mermaidで比較図を作成
- シナリオ別の選択フローチャート

### 2.3. メリット・デメリット比較表

**Agent Skillsの視点：**

| メリット | デメリット |
|---------|----------|
| 再利用性が高い | 学習が必要 |
| チーム共有が容易 | セットアップの手間 |
| カスタマイズ性 | 複雑なロジックには不向き |
| パフォーマンス | 依存関係の管理 |
| 保守性 | ベストプラクティスが未確立 |
