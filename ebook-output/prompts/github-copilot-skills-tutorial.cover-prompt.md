# ムード適用
カバーは、1つのスキル定義ファイルが評価と改善を経て育ち、チームで再利用される資産へと広がっていく様子を、学習教材らしい整理されたレイアウトで見せます。背景は明るいオフホワイトにし、コーラル・アンバー・バイオレット・ティールの温かいアクセントで、実務的な学習の安心感と設計の体系性を両立させます。

# フォント適用
モダンなサンセリフ書体を使用します。タイトルはミディアムボールドで存在感を出し、サブタイトルは軽めのウェイトで学習ガイドとしての親しみやすさを維持します。

# コンポジション
テキストコンポジション:
- タイトルとサブタイトルは上部ゾーンに配置し、イラストエリアと明確に分離する。
- タイトルゾーンは白い角丸パネルに載せ、教育書らしい整理感を演出する。
- レイアウトは上部20%をテキスト、下部80%をスキルカード・ライフサイクル・評価ゲージ・テンプレート群の複合イラストに割り当てる。

ビジュアルコンポジション:
- メインビジュアル: 中央に大きめの角丸カード（SKILL.md ファイルを象徴）。カード上部にYAML frontmatterを表す3本の細い水平線、下部に本文プレースホルダーの短いテキストラインを配置する。
- カード上部の弧に沿って、ライフサイクルの5段階（Draft → Beta → Published → Mature → Deprecated）を表す小さな円形バッジを並べ、右方向への進行を示す。
- カード下部中央から下へ矢印を伸ばし、円弧型のスコアゲージ（チェックマーク付き）につなげる。評価駆動開発（EDD）による品質評価を象徴する。
- 下段には3〜4枚の小さな角丸カードを扇状に広げて配置し、再利用可能なスキルテンプレート（コード分析・ドキュメント生成・テスト生成）を表現する。
- 背景: 薄いグリッド線（点線または実線）を敷き、教材としての構造性を演出する。暗い背景や複雑なネットワーク図は避ける。

レンダリング注記: 矩形にはクリーンなフラットフィルと細いストロークを使用し、各要素は明確な境界を持たせます。グリッドラインは薄い実線や破線で表現し、フラットベクターの美学を維持します。余計なアイコンや複雑な背景は避け、教育的で整理された印象にします。

タイプ注記: 中心のスキルカードから弧状のライフサイクルバッジ、評価ゲージ、扇状のテンプレートカード群へと視線が流れる構図を採用する。幾何学的な整理感と教育的な親しみやすさのバランスを維持する。

パレット注記: warm-creative パレットは、スキル設計・評価・運用という実務的な学習内容に温かさと整然さを与え、コーラルとアンバーが親しみやすさを、バイオレットとティールが技術的な深みと信頼感を提供する。
